This page decribes some useful linux tools and statistical programs available on rdam (and on arhus in most instances). These are very short introductions, internet is full of more thourough discussion of each of the topics presented here.

Overview

rdam and arhus are both linux (Debian and Gentoo) servers which requires some learning for new users. Furethermore, they have no scheduler and no limits are imposed on the size and the duration of the jobs that users may submit. ** With great power comes great responsibility:** you are responsible for ensuring that your jobs do not saturate or crash the servers.

  • top: monitor the server.

Linux tools

screen

  1. To create a virtual session named sname using screen enter screen -S sname. This will result in a blank command prompt, you are in the virtual session. Any job you launch in this session will continue even after you log out.
  2. To get back to the main session (detach the screen) type Ctrl-A and then d.
  3. To reattach a detached session by entering screen -r sname.
  4. To kill the sname session enter exit.

  5. To list the existing screen sessions with their status screen -ls.

    You can find much more detailed (and fancy) tutorials and tips here and here.

top

top is a useful monitoring tool. to access it, simply type top at the command line. The first few lines that appear contain server wide statistics.

  • The third line shows the share of total cpu used for:
    1. user processes
    2. system
    3. nice (low priority) processes
    4. idle
  • The fourth line shows the total, used, and available memory.

Below comes a list of every process, starting with the process id and the user it belongs as well as memory use (RES) and cpu used (%CPU).

When the free memory is low the server starts running painfully slow. rdam has a very large amount of memory nonetheless it is important that you monitor you jobs so that they do not saturate the memory.
